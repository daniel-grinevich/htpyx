Project at a Glance
Order Service Flow

The coffee ordering system starts with an iOS app where users can create, edit, view, and delete their coffee orders. Once the user finalizes their order in the cart and hits “submit order,” the details are sent asynchronously to a web API, which then stores the order in a database. The system waits for the user to scan a corresponding QR code at a Raspberry Pi station. Upon scanning, the Raspberry Pi verifies the order and triggers the coffee-making process by connecting to the Eversys coffee machine's API, which then brews the coffee according to the order specifications. This setup allows users to order and receive their coffee seamlessly with just a few taps and a QR code scan.

Challenges Faced
Issues with order lifecycle

We were experiencing issues where we would have an order placed, the espresso machine would finish the coffee, but the syrup dispenser wouldn't stop dispensing because we had no way to to track the order other than ending a confirmation response from the eversys API. This is when we implemented a state variable that we could use to track the order's lifecycle. This also solved other issues like if two people ordered at the same time or once an order is placed that the order could be cancelled until the actual QR code scan.
